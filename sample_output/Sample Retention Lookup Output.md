# Sample Retention Lookup Output

This table represents the output of the `retention_lookup` query generated by Amazon Q.
It demonstrates how the AI-assisted metadata engine identifies all records related to each student 
across multiple tables for data retention purposes.


> The retention_lookup table is then used by the data retention process to safely perform actions on each table, 
> ensuring all student-related data is properly identified.

---

## Table Columns

- `object_name` – the table containing the record  
- `attribute_name` – the column used for the relationship  
- `value` – the actual value in the child table  
- `student_id` – the ID of the student to which the record belongs
